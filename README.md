## 1. `vector`
- **概要**: 配列のような動的コンテナ
- **特徴**: メモリが連続してて、ランダムアクセスが`O(1)`で超速い
- **使い所**: 要素の追加・削除が末尾だけでOKな場合、データ数が多い場合に便利
- **例**: 画像ピクセルデータ、リストの代替

## 2. `deque`
- **概要**: 両端に要素を追加・削除できる動的配列
- **特徴**: 先頭と末尾への追加・削除が`O(1)`でできる
- **使い所**: 先頭と末尾の両方から操作する必要がある場合
- **例**: キューやデックの実装、スライドウィンドウアルゴリズム

## 3. `list`
- **概要**: 双方向リンクリスト
- **特徴**: 中間要素の追加・削除が`O(1)`でできるが、ランダムアクセスは遅い
- **使い所**: 順序が重要で、中間の要素を頻繁に追加・削除する場合
- **例**: イベントキュー、順序付きデータのバッファ

## 4. `set`
- **概要**: 重複を許さない自動ソートされた集合
- **特徴**: 重複排除が自動で、挿入・削除・検索が`O(log n)`
- **使い所**: 重複のないデータを保持し、ソートされた状態が欲しい場合
- **例**: 一意のユーザーID、ユニークな値のリスト

## 5. `unordered_set`
- **概要**: 重複なしで順序がない集合（ハッシュテーブル）
- **特徴**: 挿入・削除・検索が平均`O(1)`、順序は保証されない
- **使い所**: 重複を排除しつつ、順序不要で高速アクセスが欲しいとき
- **例**: チェック済みのURLリスト、検索済みデータの管理

## 6. `map`
- **概要**: キーと値のペアを自動ソートして保持する連想配列
- **特徴**: キーでソートされ、検索が`O(log n)`
- **使い所**: キーと値のマッピングが必要で、ソート順が重要な場合
- **例**: 辞書、電話帳、ユーザーIDと名前の対応

## 7. `unordered_map`
- **概要**: キーと値のペアを持つハッシュテーブル
- **特徴**: 挿入・検索が平均`O(1)`、順序は保証されない
- **使い所**: キーと値のマッピングが欲しいが、順序は気にしない場合
- **例**: 設定データのパース、IDとオブジェクトの紐付け
